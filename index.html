Tema 1: Fluid Web Design

Utiliza porcentajes (%) y em (Ancho de la letra M) para definir el tamaño de los elementos mostrados en pantalla. Uno de los problemas principales de este tipo de diseño es que funciona perfecto para tamaños intermedios, pero en pantallas pequeñas o pantallas grandes se pierde mucho del diseño ya que en el primer caso la informacion esta compactada y en el ultimo totalmente estirada.

Una solucion al problema del Fluid Web Design fue lo que se conoce como Liquid Web Design, el cual establece limites en los tamaños de los elementos mediante atributos como min-width:200px; max-width:600px;, pero el problema que genera esto son la gran cantidad de espacios intermedios obtenidos al trabajar en grandes pantallas y que los elementos se salen de la caja en pequeñas pantallas.

Tema 2: Adaptive Web Design

Utiliza puntos de quiebre (breakpoints) para generar diferentes vistas del contenido y como medida principal se implementan los pixeles (px). El inconveniente con este tipo de diseño es que se comienza a trabajar desde la vista de escritorio y a partir de ahi, se generan las vistas resumidas de contenido para pantallas pequeñas; si bien este tipo de diseño no esta pensado para todos los tamaños de pantalla, se adapta segun determinados parametros generales dando resultado atractivos.

El inconveniente principal con este tipo de diseño radica en la duplicidad de contenidos, ya que se generan paginas alternas para determinados tamaños de pantallas y por lo tanto se pierden conceptos del diseño al no haber sido pensadas para ese tipo de dispositivos. Otra problematica se encuentra cuando podemos cambiar entre las vistas, accediendo a paginas que no fueron pensadas para la pantalla actual.

Tema 3: Responsive Web Design

Si bien, este tipo de diseño al igual que el anterior utiliza puntos de quiebre (breakpoints), a diferencia de este implementa porcentajes (%) para los tamaños de los elementos, con maximos y minimos, segun cada cambio de tamaño. La ventaja principal de este tipo de diseño radica en el concepto One Web for All que consiste en diseñar la pagina desde el aspecto mas basico (Mobile First) y a partir de este comenzar a posicionar los elementos acorde a los diferentes breakpoints.

Este diseño recibe su nombre a partir de una publicacion realizada por Ethan Marcotte en el sitio A List Apart donde meciona las caracteristicas que debe tener un sitio web para convertirse en Responsive. Entre estas caracteristicas encontramos Multimedia flexible, un sistema de grilla y el desarrollo del contenido desde un comienzo tambien conocido como Content First

Clase 2: Grupos de Medidas

Tema 1: Medidas Relativas

Son aquellas que dependen del elemento contendor para poder definir su tamaño, es decir, son relativas a la etiqueta padre. Entre las medidas mas utilizadas encontramos porcentaje(%) y em, la primera aplicada principalmente para los bloques de contenido y la otra para las tipografias. Por lo tanto, una etiqueta con width:80% no medira lo mismo, si se encuentra dentro de otra que posea un tamaño de 2000px, a otra que posea 1000px.

  • %: Porcentaje del tamaño del elemento contenedor
  • em: Ancho de la letra "M" de la etiqueta padre
  • ex: Alto de la letra "x" del elemento padre
  • ch: Ancho del numero "0" de la etiqueta contenedora

Tema 2: Medidas Absolutas

Son aquellas que independientemente del contexto, siempre van a representar un valor universal. Si bien, no se suelen utilizar demasiado, la medida de este tipo que mas se implementa son los pixeles(px), que se consideran una medida hibrida ya que dependen de la resolucion para reflejar un tamaño final, pero la cantidad de puntos en pantalla, siempre es la misma.

  • cm: medida base 1/100m
  • in: medida 2.54cm
  • pt: fuente 1/72in
  • px: puntos en pantalla (resolucion)

Tema 3: Medidas Flexibles

Son aquellas que utilizan como referente el tamaño del area visible de la ventana del navegador(VIEWPORT). Si bien se suelen considerar relativas, a diferencia de la mayoria estas No dependen del contenedor, sino que dependen de la misma ventana. Cabe destacar que no necesariamente el tamaño de la ventaña y la escala de la pagina deben ser las mismas.

  • vw:ancho del viewport
  • vh:alto del viewport
  • vmin:medida mas pequeña del viewport
  • vmax:medida mas grande del viewport
